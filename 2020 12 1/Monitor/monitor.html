Bounded-Buffer Problem Using a Monitor

This Java applet demonstrates the bounded-buffer consumer/producer problem using monitor. A monitor is a software module consisting of one or more procedures, and initialization, and local data. The data variables in the monitor can be accessed by only one process at a time; thereby maintaining mutual exclusion. This means that producer and consumer cannot simultaneously access the buffer. A monitor provides synchronization by the use of condition variables; notFull and notEmpty in this applet. wait(cv) suspends the execution of the calling process on condition cv. Another process can now use the monitor. notify(cv) resumes execution of the process that is suspended after a wait on the same condition.
